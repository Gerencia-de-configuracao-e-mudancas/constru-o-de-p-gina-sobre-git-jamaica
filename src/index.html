Comandos Basicos do GIT

Usar o git pela linha de comando pode parecer meio complicado. Para te ajudar com isso, nós decidimos criar uma lista com alguns comandos basicos do Git, vamos lhe dizer oque cada um faz e como usalo. Esperamos que quando você termine de ler este guia, esteja apto para trabalhar em empresas como a RedeTV, SkyGato, etc.

Trabalhando com repositórios Locais

git init

O comando git init é usado para transformar um diretório comum em um repositório Git vazio. Este é o primeiro passo para criar um repositório onde você poderá versionar seu código.

Uso:

# Navegue até o diretório onde deseja iniciar o repositório:
$ cd caminho/código
          
# Inicialize o repositório Git no diretório:
$ git init

Na prática:

# Indo até o repositório do projeto
$ cd ~/site
          
# Inicializando um repositório git nesse diretório
$ git init
Repositório Git vazio inicializado em ~/site/.git/

git add

O comando git add é como colocar seus arquivos em uma bandeja de preparação antes de confirmá-los no Git. Você coloca os arquivos na bandeja um por um ou todos de uma vez. É uma etapa necessária antes de salvar as mudanças no Git. Isso ajuda a escolher quais alterações você deseja incluir na próxima "salvação". Em resumo, o git add coloca seus arquivos na fila para serem confirmados no Git.

Uso:

$ git add <nome do arquivo ou diretório>

Na Prática:

# Para adicionar todos os arquivos não preparados:
$ git add .
          
# Para adicionar um arquivo específico:
$ git add index.html
          
# Para adicionar um diretório inteiro:
$ git add css

git commit

O comando git commit é como salvar um ponto de verificação das mudanças que você fez em seus arquivos. Cada ponto de verificação tem um número único. É importante adicionar uma mensagem a cada ponto de verificação para dizer o que foi feito. Isso ajuda a lembrar ou explicar as mudanças. Em resumo, git commit é como salvar as mudanças no Git, e a mensagem ajuda a entender o que foi feito.

Uso:

# Adicionando um commit com mensagem
$ git commit -m "Meu Primeiro Commit"

Na Prática:

$ git commit -m "Meu Primeiro Commit"
[main (root-commit) 919941c] Meu Primeiro Commit
1 file changed, 0 insertions(+), 0 deletions(-)
create mode 100644 index.html

git status

O comando git status fornece informações sobre o estado atual do seu repositório Git. Ele mostra o ramo em que você está trabalhando e se há arquivos modificados. Se houver arquivos modificados, ele informará se essas alterações foram ou não preparadas para uma confirmação. Em resumo, o git status ajuda a manter o controle das mudanças em seu projeto Git e indica se você precisa tomar medidas, como preparar e confirmar as alterações.

Uso:

$ git status

Na Prática:

# Mensagem quando os arquivos não foram preparados (git add)
$ git status
No ramo main
Arquivos não monitorados:
    (utilize "git add ..." para incluir o que será submetido)
            index.txt
              
nada adicionado ao envio mas arquivos não registrados estão presentes (use "git add" to registrar)
          
# Mensagem quando os arquivos foram preparados, mas não confirmados (git commit)
$ git status
No ramo main
Mudanças a serem submetidas:
      (use "git restore --staged ..." to unstage)
	         new file:   z.txt

# Mensagem quando todos os arquivos foram preparados e confirmados
$ git status
No ramo main
nothing to commit, working tree clean

git config

O comando git config permite que você ajuste configurações importantes no Git, como seu nome de usuário (user.name) e endereço de email (user.email). Essas configurações determinam como seus commits são identificados em seu próprio computador.

Uso:

$ git config <setting> <command>

Na Prática:

# Configurando globalmente (para todos os seus repositórios Git)
$ git config --global user.email "erick@only.com"
$ git config --global user.name "Erick Ribeiro"

# Configurando localmente (apenas para o repositório atual)
$ git config user.email "erick@only.com"
$ git config user.name "Erick Ribeiro"

git branch

O comando git branch é usado para verificar em qual branch o repositório local está, criar um nova branch ou excluir uma branch existente.

Uso:

# Cria uma nova branch
$ git branch <nome_branch>

# Lista as branches locais e remotas
$ git branch -a

# Apaga uma branch
$ git branch -d <nome_branch>

Na Prática:

# Criando uma nova branch
$ git branch new_feature

# Listando as branches
$ git branch -a
* main
  new_feature
  
# Apagando uma branch
$ git branch -d new_feature
Deleted branch new_feature (was 0254c3d).

git checkout

O comando git checkout é usado para começar a trabalhar em uma branch diferente no Git. Ele permite que você mude o foco do seu trabalho para outra branch.

Uso:

# Muda o para uma branch existente
$ git checkout <nome_branch>

# Cria uma branch e muda imediatamente para ela
$ git checkout -b <nome_branch>

Na Prática:

# Mudando para a branch 'new_feature'
$ git checkout new_feature
Switched to branch 'new_feature'

# Criando a branch 'staging' e mudando para ela
$ git checkout -b staging
Switched to a new branch 'staging'

git merge

O comando git merge é usado para combinar as mudanças de uma branch específica na branch atual. Por exemplo, você pode usar isso para incorporar as alterações feitas em uma branch de desenvolvimento em sua branch principal.

Uso:

# Mescla as alterações da branch especificada na branch atual
$ git merge <branch_name>

Na Prática:

# Mesclando as alterações da branch 'new feature' na branch atual
$ git merge new_feature
Updating ed95ac5..453f8a6
Fast-forward
 mataGrama.exe | 0
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 mataGrama.exe
